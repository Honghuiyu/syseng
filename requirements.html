MoSCoW Report

Must have

(These are the absolute minimum requirements that we will strive to meet during this project)

  • Cross-platform app â€“ it must work on both Android and iOS
  • C# Xamarin code-base
  • Geolocation features to locate the user, possibly using Bing Maps API
  • Integration with the Recommendation Engine/API
  • Database of landmarks/ places of attraction- have it running, not build (Azure table storage)

Should have:

(Once we have implemented the Must haves, we will focus on these primarily)

  • A clean User Interface
  • Ability to push notifications
  • An interface for Kent County council to add/approve places of interest
  • A weather checking function

Could have:

(If we had spare time on the project once we have implemented all Must haves and most Should haves, we will try and include the following features)

  • Support for wearables
  • Battery Optimization
  • Be an extension of the existing project
  • A method of tuning the frequency of the push notifications received by the user
  • Recommendation feature that can be used to plan ahead
  • Calendar integration
  • Check + decrease/increase radius depending on location + no of points of interest

Won't (necessarily) have:

The following won't be an expectation from this project

  • Bot integration to make it more interactive
  • A description of the attraction or ability to expands into a full page of information
  • Integration with beacons
  • Any additional machine learning
  • Windows phone support
