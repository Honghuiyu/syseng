Changes in MoSCoW report

  • - Our initial brief was to create a tourist 'layer' which gaave us the option to create 2 things: an app or a bot. After conducting a heavy amount of research, through weighing the pros and cons of each, we decided to create the app.
  • - Initially we were planning to use the reccomendation engine and build on top of it. However, since we ended up spending too much time trying to get it to work, we had to move on from it and focus on actually creating an app.

Use case: Push notifications Geolocation Weather checker
ID: - -
Brief description The main feature of the application that alerts the user when they are walking past a historical monument. Geolocation feature to detect the location of the user User had the option to check the weather using a button
Primary actors The phone The user Users
Secondary actors The application The application -
Preconditions (the system state before the use case can begin)
  • Connected to Wi-Fi or active network
  • Geolocation can be tracked and pinpointed to a specific point
  • The user need to have GPS enabled
  • Must be connected to internet/wifi on their mobile device
 Must be connected to internet/wifi on their mobile device
Main flow (the actual steps of the use case)
  • Location noted
  • Retrieve information about the surroundings from the database
  • User location detected
  • This information is used to retrieve information from the database.
  • The user location is used as a filter criteria.
  • The app will show all modules that satisfy the filter.
  • The user can scroll through list of available options (of places to visit).
  • Use case initiated when the user clicks the 'check weather' button.
  • API call requested and returns with realtime weather data.
  • Data displayed to user.
Postconditions (the system state when the use case is finished) Notification pushed to the user The modules are displays as per the parameters of the filter Shows weather for the day
Alternative flows
  • Calendar indicates that the user is currently busy so the notification is not sent
  • Wifi connection lost so notification is not pushed- hopefully deleted
  • Frequency is set to a minimum so the notification is deleted if a certain number of notifications have already been pushed
  • Location not found
  • No places to visit nearby
  • Widen the radius of search until app is able to show modules
